Password Policy & Authentication Guidelines

Introduction

This page provides an overview of modern password policies and authentication guidelines in line with OWASP, NIST, and other leading security frameworks.

Password Length Requirements

  • Minimum password length should be at least 12 characters.
  • Maximum password length should allow at least 64 characters, rejecting anything over 128 characters.
  • Passwords shorter than 8 characters are considered weak (NIST SP800-63B).

Character Support & Rules

  • Allow any printable Unicode characters, including spaces and emojis.
  • Do not truncate passwords silently if they exceed the maximum length.
  • No forced password composition rules (no requirements for uppercase, lowercase, numbers, or special characters).
  • Support copy-paste, browser password managers, and external password managers.

Password Strength & Breach Detection

  • Include a password strength meter to assist users in creating stronger passwords. Libraries like zxcvbn-ts can be used.
  • Passwords should be checked against a database of breached passwords. Services like Pwned Passwords can be used, either via API or self-hosted solutions.
  • Zero-knowledge proof mechanisms should be used when interacting with external APIs to check passwords.

Password Change & Rotation

  • Users must be able to change their passwords at any time.
  • Password change must require the current password along with the new one.

Usability Considerations

  • Allow users to view the entire masked password or temporarily view the last typed character if the platform does not provide this feature natively.
  • Support for longer passphrases improves usability and encourages better password practices.

Best Practices from OWASP ASVS 4.0

  • Minimum password length: 12 characters.
  • Maximum password length: 64 characters (up to 128 can be rejected).
  • Do not perform password truncation.
  • Allow Unicode characters including emojis and spaces.
  • Provide password breach checks and strength meters.
  • No requirements on character composition.
  • Do not enforce password expiration policies without cause.
